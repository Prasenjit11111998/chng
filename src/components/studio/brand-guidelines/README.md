# Studio: Brand Guidelines Generator

The **Brand Guidelines Generator** is a flagship feature within the Chng Studio suite. It allows users to input their brand's foundational elements—logos, colors, typography, and brand identity details—and export a professional, print-ready, multi-page PDF brand book.

## Architecture

The module is broken down into three main domains:
1. **Interactive Wizard (`/wizard`)**: The form interface where users input data step-by-step.
2. **Live HTML Preview (`/preview`)**: An interactive, scaled-down DOM view showing what the final output will look like as the user edits in real-time.
3. **PDF Generation Engine (`/pdf`)**: Uses `@react-pdf/renderer` to draw the layout to a canvas and construct a downloadable PDF binary.

### Key Components

- `BrandGuidelinesStudio.tsx`: The main orchestration file holding state.
- `LivePreviewPanel.tsx`: The right-hand pane showing the visual feedback.
- `PDFDocument.tsx`: The core React-PDF document structure. Contains all the complex layout math, grid alignments, and styling specific to PDF rendering.

## Core Features

- **Step-by-Step Data Entry:** Collects Brand Info, Logos (Primary, Light, Dark, Icon), Colors (HEX with auto CMYK/RGB conversion), and Typography selection.
- **Dynamic Theming:** Offers `Light` and `Dark` theme templates for the PDF output, adhering to "Studio Hornbill" bold aesthetics.
- **Auto Logo Inversion:** Intelligently pairs dark logos with light backgrounds and light logos with dark backgrounds.
- **Custom & Google Font Support:**
  - Standard Google fonts are resolved dynamically as `.ttf` files via direct download endpoints (`gwfh.mranftl.com`) since `@react-pdf/renderer` does not support standard WOFF2 web API endpoints.
  - Users can upload `.ttf` files via an `<input type="file">`, which are converted to Data URIs (base64) so they can be injected synchronously into the PDF rendering pipeline without network latency issues.

## Design System (Impeccable Standards)

The layout generated by this feature has been polished to meet premium agency standards:
- **Binder Edge Bleed:** A bold `16px` accent color bar runs along the left edge of interior pages, simulating a printed spine binding.
- **Whitespace & Scale:** Uses a rigid spacing scale (`SP.xs` through `SP.xxl`). Headings are dramatically scaled (`112px` font specimens) with tight kerning (`-2.5px` on huge displays) to create visual tension.
- **Object Containment:** Logos use `objectFit: 'contain'` with tightly defined aspect-ratio boxes to prevent layout drift or skewing regardless of user upload size.
- **High-Contrast Typography:** Automatic luminance checking ensures text color on brand-colored backgrounds never drops below WCAG accessibility thresholds.

## Important Note on Fonts for PDF Export

When modifying `@react-pdf/renderer` styles, **never** rely on standard web CSS fonts for the PDF generation.
1. The DOM Preview can use standard web fonts.
2. The PDF generator **must** have access to the raw `.ttf` binaries. 
3. Any font used in `PDFDocument.tsx` MUST be explicitly registered via `Font.register({ ... })` before rendering starts.
